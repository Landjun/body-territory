# 作品说明｜这个项目展示了我什么

> 给面试官 / HR / 访客的快速说明页。读完这一页,你会知道这个仓库不是"跑步资料夹",而是一个能力样本。

## 一句话

我用费曼学习法 + GitHub 工程化工作流,把一个模糊的目标(安全地恢复跑步、重建身体)拆解成一个**可量化、可追踪、可自动化维护的个人系统**,并长期运营它。

## 我解决的问题

受伤后恢复训练反复受挫,根本原因不是"不够努力",而是缺一套**决策系统**:今天能不能跑?疼到几分要停?状态差怎么降级而不断档?学到的东西怎么沉淀成能力?

我把它做成了一个九模块系统,把"凭感觉"变成"按规则 + 有记录 + 可复盘"。

## 系统架构

```text
身份定位 → 跑步安全 → 力量重建 → 伤病复盘 → 训练日志 → ACE-CPT → 费曼输出 → 品牌沉淀 → 自动化维护
   ↑ 为什么练      ↑ 决策系统(跑前5问/疼痛分级/A-E等级)      ↑ 学习方法论       ↑ 让系统不腐化
```

- **知识层** `docs/`:9 大模块,每篇都含 目的 / 原则 / 最小行动 / 费曼自检 / 待补充。
- **工具层** `scripts/`:4 个纯标准库脚本(进度统计、训练日志生成、费曼笔记生成、金句卡片生成)。
- **工程层** `.github/`:3 条 GitHub Actions(断链检查、进度校验、卡片自动生成)+ Issue/PR 模板。
- **证据层** `logs/` + 进度看板 + 徽章:把训练和学习变成可追踪的长期证据。

## 技术栈

- **Git / GitHub**:版本管理、Actions(CI)、Issue/PR 模板、动态徽章(shields.io endpoint)。
- **Python(仅标准库)**:零依赖、跨平台的自动化脚本。
- **Markdown + SVG**:结构化知识 + 可版本管理的视觉素材。

## 几个值得看的细节

- **进度是真实的,不伪造**:当前 0%,所有任务都是真实待办;脚本会过滤掉"标题行"避免虚高计数。
- **自动同步**:跑一次 `update_progress.py`,README 看板、8 个模块小计、顶部徽章三处一起更新。
- **自我维护**:CI 会在进度看板过期或出现断链时报警,改了金句生成器会自动重生成卡片。
- **不医疗化**:涉及疼痛/伤病的内容均带免责声明,只做个人记录与决策,不做诊断。
