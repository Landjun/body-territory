# 手表录音笔工作流：把费曼输出变成零成本启动

## 本页目的

记录我的录音笔 + 手表工作流，把"开口讲"的启动成本降到最低。

## 工作流

- 录音笔夹在手表上；
- 像拿麦克风；
- 一抬手就能讲；
- 不用打开软件；
- 启动成本低；
- 适合训练前判断；
- 适合训练后复盘；
- 适合散步时讲知识点；
- 输出后转成文字；
- 整理为 GitHub 笔记。

## 为什么有效

把工具固定在手腕上，"想讲"和"开始讲"之间几乎没有摩擦。摩擦越小，输出越频繁，费曼闭环越容易跑起来。

## 适用场景

- 训练前：口述[跑前 5 问](../02-running-safety/pre-run-checklist.md)；
- 训练后：口述当天复盘；
- 快走/散步时：讲一个知识点；
- 学习卡住时：把卡点讲出来。

## 从录音到证据

```text
抬手就讲 → 录音 → 转文字 → 整理成笔记 → 提交到 GitHub
```

## 最小行动

今天训练前，抬手对着录音笔讲 1 分钟"我现在的身体状态和今天的计划"。

## 费曼讲解问题

- 这套录音工作流用一句话怎么讲？
- 为什么"降低启动成本"对输出这么关键？
- 它如何帮助我减少受伤（通过更好的训练前判断）？
- 它如何服务网络安全主线？
- 今天我能做的最小动作是什么？
