1. 直接对着标题开始讲；
2. 讲不清楚的地方标记为卡点；
3. 回到资料补学；
4. 再讲一遍；
5. 形成 3 分钟讲稿；
6. 扩展成 10 分钟讲稿；
7. 沉淀为笔记 / 公众号 / 训练卡片 / 作品集证据。

## 为什么先讲

- 先讲不是偷懒；
- 先讲是暴露卡点；
- 卡点比完整笔记更重要；
- 讲出来才知道自己会不会。

## 我的学习特点

我是"演讲型人格"，适合直接讲：看着书、课程、PPT、目录、H1 标题就能开始讲，越讲思维越顺；不适合"先完整整理再输出"。所以我的最佳路径是：

```text
边看边讲 → 讲中暴露卡点 → 卡点驱动补学 → 再讲一遍 → 沉淀证据
```
