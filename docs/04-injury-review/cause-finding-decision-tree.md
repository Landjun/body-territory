# 疼痛诱因排查逻辑树

> 个人训练复盘方法,非医疗诊断。任何时候出现[红旗信号](../02-running-safety/red-flags.md)都先停、必要时就医,再谈排查。

## 本页目的

把"我为什么疼"从一团乱麻,变成一条**可一步步走下来的排查流程**。它不告诉你"是什么病"(那是医生的事),而是帮你定位**可控的训练诱因**,做出降级和调整决定。

## 怎么用

按顺序从 Step 0 往下走,每一步记录答案。走完你会得到:一个诱因假设 + 一个今日决策 + 一个回归条件。

## 排查流程

```text
Step 0  安全先行
        是否有红旗信号?(疼痛5+ / 步态改变 / 肿胀 / 麻木 / 休息不缓解 / 尖锐刺痛)
          ├─ 是 → 停止跑跳,必要时就医。排查到此为止,先处理安全。
          └─ 否 → 继续 Step 1

Step 1  定位与描述
        位置(右踝外侧?小腿中段?膝前?) + 性质(酸/紧/钝/刺) + 程度(0-10)
        → 记录基线,便于追踪趋势

Step 2  时间线(最关键的一问)
        这个不适是什么时候的?
          ├─ 跑前就有     → 多半是上一轮负荷未恢复 / 慢性紧张 → 重点查 Step5 恢复 + Step3 负荷
          ├─ 跑中出现     → 越跑越重?是 → 偏危险,当次就该停;热身后减轻?相对安全但仍查 Step3/4
          ├─ 仅跑后       → 查 Step3 负荷是否偏大 + 冷身/拉伸是否缺位
          └─ 次日才加重   → 上一次负荷过量的典型信号 → 降级 + 重点查 Step3

Step 3  负荷排查(过用伤的头号嫌疑)
        最近 7-14 天有没有:
          - 跑量/时间骤增?(违背渐进,见 负荷管理)
          - 高强度(间歇/节奏)突然变多?
          - 频率变密、恢复日变少?
          - 跑步机/鞋/地面/坡度 有变化?
        命中越多 → 诱因越可能在"负荷骤增或条件改变"

Step 4  力量与结构(承受力一侧)
        对应部位的上下游是否薄弱?
          - 膝不适 → 查 臀(髋稳定) + 踝(背屈/稳定)
          - 小腿/跟腱 → 查 提踵力量/离心 + 踝活动度 + 跑量
          - 右踝 → 查 踝稳定/活动度 + 小腿
        薄弱 → 把对应力量项纳入恢复期重点

Step 5  恢复背景(放大器)
        睡眠是否不足?压力是否大?整体疲劳?
        → 这些会降低"承受力",让同样负荷变成超载 → 需整体降级

Step 6  形成决策
        综合上面:
          - 疼痛 3-4 或次日加重 → 不跑,改快走+康复力量+拉伸,写复盘
          - 疼痛 5+ / 步态改变 → 停跑跳,必要时就医
          - 找到诱因 → 针对性调整(减量 / 补力量 / 换条件 / 加恢复)
          - 回归条件 → 见 回归跑步清单
```

## 把结论写进复盘

走完这棵树,用[伤病日志模板](injury-log-template.md)或对应部位复盘([右脚踝](ankle-pain-review.md) / [小腿](calf-tightness-review.md))记录:诱因假设、今日决策、回归条件。下次同样的疼再来时,先翻这条记录。

## 核心原则

- **安全 > 排查 > 训练**:红旗信号面前不排查,先处理安全;
- **过用伤先查负荷,再查力量,再查恢复**;
- 排查的产出是"可控诱因 + 决策",不是"诊断病名"。

## 费曼讲解问题

- 这棵树为什么把"红旗信号"放在 Step 0?
- "时间线(跑前/跑中/跑后/次日)"为什么是最关键的一问?
- 过用性疼痛,应该按什么顺序排查(负荷/力量/恢复)?
- 它和网络安全的"先止损、再定位根因"有什么相通?
- 今天我能做的最小动作是什么?
